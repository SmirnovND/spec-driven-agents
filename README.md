# Spec-Driven Agents

Этот репозиторий описывает подход к разработке программного обеспечения,
в котором **Markdown-спецификации являются основным источником истины**,
а код — их реализацией.

Подход ориентирован на:
- работу с ИИ-агентами (Zencoder, Cursor, ChatGPT и др.)
- поддержку живой документации
- строгую трассируемость изменений
- снижение рассинхронизации между бизнес-логикой и кодом

---

## Основная идея

- Каждому Go-файлу соответствует Markdown-спецификация с тем же именем
- Спецификации описывают бизнес-логику и правила
- ИИ-агент работает **не с кодом напрямую**, а через спецификации
- Любое изменение начинается с обновления спеки

---

## Как использовать с AI-агентом

1. Создайте кастомного агента (например, в Zencoder)
2. Скопируйте содержимое `agent_prompt.md` в system prompt агента
3. Убедитесь, что файл `spec_rules.md` доступен агенту в репозитории
4. В задачах агенту указывайте входную спецификацию (entry point)
